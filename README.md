# Weezer-TD-Engine 🚀

**Weezer-TD-Engine** — это модульный игровой движок и редактор уровней для игр в жанре **Tower Defence (защита башен)**, разработанный на платформе .NET 9. Он поддерживает одиночную игру, многопользовательский режим по локальной сети и продвинутую систему динамических модификаций через динамические библиотеки (DLL).

---

## 🛠️ Архитектура и структура проекта

Проект разделен на три ключевых модуля:
1. **`shared_simulation_engine` (Shared Simulation Engine)**:
   Ядро игрового процесса. Содержит всю математику, логику симуляции врагов, башен, снарядов, обработку игрового поля, сетевые протоколы и систему синхронизации.
2. **`Weezer-Tower-Defence` (Игровой клиент)**:
   Визуальный клиент на базе MonoGame. Отвечает за рендеринг графики, воспроизведение звуков, интерфейс пользователя (UI) во время игры и обработку ввода.
3. **`Weezer-Tower-Defence-Editor` (Редактор уровней)**:
   Инструмент для визуального проектирования карт, настройки путей движения врагов, параметров волн, начального баланса и тестирования созданных конфигураций.

## 🔌 Система Модификаций (Создание модов в виде DLL)

Движок разработан с акцентом на расширяемость. Вы можете создавать собственные типы башен, уникальные снаряды или новые типы врагов, компилируя их в `.dll` файлы.

### Как работает загрузка модов:
- Движок сканирует папки `/damageDealers`, `/enemies` и `/towers` (пути настраиваются через класс `PathService`).
- Каждый мод состоит из двух компонентов:
  1. **`.dll` файл**: Компилируемая сборка на C#, содержащая классы, реализующие интерфейсы движка (например, `ITowerBehavior`, `IEnemyType`, `StandardBulletBehavior`).
  2. **`.json` конфигурация**: Описывает метаданные мода: имя отображения (`Name`), имя класса (`ClassName`), имя DLL-файла (`FileName`), а также аргументы инициализации (`Args` со свойствами `Name` и `Type`).

### Пример создания мода для башни:
1. Создайте библиотеку классов C# (.NET 9) и добавьте ссылку на `shared_simulation_engine.dll`.
2. Реализуйте интерфейс `ITowerBehavior` или унаследуйте класс от `DefinitionTowerBehavior`.
3. Скомпилируйте DLL и поместите её в папку плагинов башен.
4. Создайте JSON-файл конфигурации, соответствующий классу:
   ```json
   {
     "name": "Custom Laser Tower",
     "className": "LaserTowerBehavior",
     "fileName": "LaserTower",
     "args": [
       { "name": "damage", "type": "float" },
       { "name": "fireRate", "type": "float" }
     ]
   }
   ```
5. Движок автоматически загрузит сборку через `Assembly.Load` / `Assembly.LoadFrom`, свяжет параметры из JSON и сделает башню доступной для покупки в игре и выбора в редакторе!

## 🌐 Локальная сеть (LAN) и Raft Consensus (Переключение хоста)

Движок поддерживает полноценный мультиплеер по локальной сети с автоматическим поиском лобби и надежным протоколом синхронизации.

### 📶 Автоматическое обнаружение лобби (UDP Broadcast)
Благодаря `UdpLobbyDiscovery` игрокам не нужно вводить IP-адрес вручную. Хост лобби периодически рассылает UDP-широковещательные анонсы на порт `47777`. Клиенты слушают сеть, автоматически формируют список доступных игр и отображают статус готовности участников в реальном времени.

### 👑 Отказоустойчивость сети и алгоритм Raft (Host Switching)
Если создатель игры (хост) внезапно отключается, вылетает или теряет связь, игра **не прерывается**. В движок встроен распределенный консенсус **Raft** (`RaftGameNode` на UDP-порту `47780`):
- Игроки обмениваются пакетами состояния игры (Heartbeats).
- Если клиенты не получают данные от хоста в течение **3 секунд** (`HeartbeatTimeout`), они запускают процедуру проверки доступности пиров.
- Если пиры доступны, автоматически инициируется голосование (Raft Election).
- Выбранный нодой «Лидер» становится новым хостом лобби.
- Игра плавно переключает симуляцию: новый хост берет на себя управление волнами и спавном врагов, а остальные клиенты на лету переподключаются к его IP-адресу.
- Для игрока это выглядит как кратковременное появление оверлея «Соединение потеряно», после чего игра продолжается с того же момента.

## 🚀 Сборка и запуск

Для сборки проекта вам понадобится **.NET 9.0 SDK**.

### Компиляция через CLI:
```bash
# Сборка всего решения
dotnet build
```

### Запуск игры:
```bash
cd Weezer-Tower-Defence
dotnet run
```

### Запуск редактора:
```bash
cd Weezer-Tower-Defence-Editor
dotnet run
```
