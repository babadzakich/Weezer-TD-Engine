# 🛰 Weezer Net-Shaper (P2P Proxy)

**Weezer Net-Shaper** — это специализированный прокси-сервер на Python, разработанный для тестирования сетевой устойчивости **Weezer Tower Defence Engine**. Он выступает в роли MITM (Man-in-the-Middle) узла, позволяя эмулировать реальные сетевые проблемы (задержки, потери пакетов, разделение сети) в P2P-сессиях.

## 🚀 Основные возможности

- **Async UDP Routing:** Высокопроизводительная пересылка пакетов с использованием `asyncio`.
- **Chaos Middleware:**
    - **Latency & Jitter:** Эмуляция высокого пинга и нестабильного соединения.
    - **Packet Loss:** Настраиваемая вероятность потери данных.
    - **Duplication:** Проверка идемпотентности логики игры через дублирование пакетов.
- **Advanced P2P Scenarios:**
    - **Network Partition (Split Brain):** Изоляция групп игроков друг от друга для тестирования слияния сессий.
    - **Host Sabotage:** Автоматическое обнаружение хоста и постепенное ухудшение его связи для провокации перевыборов (Host Election).
- **Runtime Control:** Интерактивная CLI-консоль для изменения параметров сети "на лету".
- **Hot Reload:** Автоматическое применение настроек из `rules.json` без перезагрузки сервера.

## 🛠 Технический стек

- **Python 3.11+**
- **Asyncio** (для сетевого взаимодействия)
- **Aioconsole** (для интерактивного CLI)
- **JSON Protocol** (совместим со спецификацией Weezer TD)

## 📦 Установка

1. Убедитесь, что у вас установлен Python 3.11 или выше.
2. Активируйте виртуальное окружение:
   ```bash
   .venv\Scripts\activate  # Windows
   source .venv/bin/activate  # Linux/macOS
   ```
3. Установите зависимости:
   ```bash
   pip install aioconsole
   ```

## 🎮 Запуск

### 1. Запуск Прокси
Прокси настраивается через аргумент `--nodes`, где указывается список узлов в формате `ID:RealIP:RealPort:ProxyPort`:

```bash
python proxy.py --nodes "1:127.0.0.1:6001:5001 2:127.0.0.1:6002:5002"
```
*В этом примере:*
- Игрок 1 слушает на `6001`, но отправляет данные на прокси-порт `5001`.
- Игрок 2 слушает на `6002`, но отправляет данные на прокси-порт `5002`.

### 2. Запуск Тестовых Клиентов (Mock Peers)
Для проверки работы без основной игры можно использовать `mock_peer.py`:

**Терминал 1 (Игрок 1):**
```bash
python mock_peer.py --id 1 --port 6001 --proxy-addr 127.0.0.1:5001 --target-id 2
```

**Терминал 2 (Игрок 2):**
```bash
python mock_peer.py --id 2 --port 6002 --proxy-addr 127.0.0.1:5002 --target-id 1
```

### 3. Запуск НАСТОЯЩЕЙ игры через прокси

Игра умеет «подменять сокеты» и гнать весь внутриигровой трафик (состояние, запросы клиента, Raft-консенсус) через прокси. Включается переменными окружения — на обычную игру по LAN это никак не влияет.

| Переменная | Описание | Пример |
| :--- | :--- | :--- |
| `WTD_PROXY` | включить режим прокси | `1` |
| `WTD_PROXY_ID` | целочисленный id узла (как в `--nodes`) | `1` |
| `WTD_PROXY_SEND` | `ip:port` прокси-порта, **куда** слать | `127.0.0.1:5001` |
| `WTD_PROXY_BIND` | реальный порт, **где** слушать | `6001` |

Запусти прокси и два экземпляра игры (форма `env VAR=val …` работает и в bash, и в fish):

```bash
# Терминал 1 — прокси (узлы 1 и 2)
./run_game_proxy.sh
# (либо вручную: python proxy.py --nodes "1:127.0.0.1:6001:5001 2:127.0.0.1:6002:5002")

# Терминал 2 — игра, узел 1
env WTD_PROXY=1 WTD_PROXY_ID=1 WTD_PROXY_SEND=127.0.0.1:5001 WTD_PROXY_BIND=6001 \
    dotnet run --project ../Weezer-Tower-Defence/Weezer-Tower-Defence-Game.csproj

# Терминал 3 — игра, узел 2
env WTD_PROXY=1 WTD_PROXY_ID=2 WTD_PROXY_SEND=127.0.0.1:5002 WTD_PROXY_BIND=6002 \
    dotnet run --project ../Weezer-Tower-Defence/Weezer-Tower-Defence-Game.csproj
```

Оба экземпляра на одной машине находят друг друга через обычный loopback-дискавери (UDP 47777) и собирают лобби; дальше весь игровой трафик идёт через прокси. Захости лобби на узле 1, подключись узлом 2, обозначь готовность и стартуй. Потом из консоли `proxy >` крути хаос: `set_delay 1 300`, `drop 1 0.3`, `sabotage_host`, `split_groups 1 2`, `status`.

> При старте игра логирует `[ProxyLink] ENABLED node=… send->… bind=…` — значит трафик пошёл через прокси.

## 🕹 Команды управления (CLI)

После запуска прокси вы увидите приглашение `proxy > `. Доступны следующие команды:

| Команда | Описание |
| :--- | :--- |
| `status` | Показать таблицу маршрутизации и активные правила. |
| `set_delay <id> <ms>` | Установить задержку (пинг) для конкретного игрока. |
| `drop <id> <chance>` | Установить шанс потери пакетов (от 0.0 до 1.0). |
| `isolate <id>` | Полностью заблокировать трафик от игрока. |
| `unisolate <id>` | Снять блокировку трафика с игрока. |
| `split_groups 1,2 3,4` | Разделить игроков на изолированные группы (Partition). |
| `sabotage_host` | Найти хоста по трафику и начать плавно душить его канал. |
| `help` | Список всех команд. |

## 📄 Конфигурация (rules.json)

Вы можете заранее прописать правила в `rules.json`. Прокси подхватит их автоматически при старте или изменении файла:

```json
{
  "defaults": {
    "latency": 50,
    "jitter": 10,
    "drop_rate": 0.05
  },
  "nodes": {
    "1": { "latency": 200 }
  }
}
```

*Разработано для Weezer Tower Defence Engine.*
